1. Line 29 — `String.format` uses implicit default locale.
2. Line 37 — `String.format` uses implicit default locale.
3. Line 104 — `fun isRunning()` is never used.
4. Line 157 — `String.format` uses implicit default locale.
5. Line 165 — `String.format` uses implicit default locale.
6. Line 195 — `files.isNotEmpty()` is always true when reached (redundant condition).

(That's 6 distinct issues; the IDE counts 7 problems because the locale warning
covers 4 separate `String.format` calls.)

## Files to change
- `app/src/main/java/in/sreerajp/chronotune_smart_clock/data/Models.kt`

## Fix
1. Add `import java.util.Locale`.
2. Pass `Locale.getDefault()` as the first arg to all four `String.format(...)`
   calls (lines 29, 37, 157, 165). Using `Locale.getDefault()` makes the existing
   implicit behavior explicit — no behavior change, just silences the lint.
3. Remove the unused `fun isRunning(): Boolean = state == STATE_RUNNING` (line 104).
   Verified unused across the codebase (other `isRunning` hits are unrelated local
   vals in StopwatchScreen.kt).
4. Simplify the `describeTracks()` branch at line 195 from
   `ambients.isEmpty() && files.isNotEmpty() ->` to `ambients.isEmpty() ->`.
   When this branch is reached, an empty `ambients` already implies a non-empty
   `files` (the earlier `ambients.isEmpty() && files.isEmpty()` branch ruled out
   the empty case), so the second condition is always true. Behavior unchanged.
