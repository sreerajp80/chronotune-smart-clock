# Change log: Fix lint warnings in Models.kt

Implements plan `plans/20260628_000100_fix-models-lint.md`.

## Changes
All in `app/src/main/java/in/sreerajp/chronotune_smart_clock/data/Models.kt`:

1. Added `import java.util.Locale`.
2. Made locale explicit in all four `String.format(...)` calls (in
   `Alarm.getFormattedTime` and `MusicSchedule.getFormattedTime`) by passing
   `Locale.getDefault()` as the first argument. This preserves the prior implicit
   default-locale behavior while silencing the "implicitly using the default
   locale" lint warning.
3. Removed the unused `TimerItem.isRunning()` function. Confirmed unused across
   the codebase (other `isRunning` references are unrelated local variables in
   StopwatchScreen.kt).
4. Simplified the `MusicSchedule.describeTracks()` `when` branch from
   `ambients.isEmpty() && files.isNotEmpty()` to `ambients.isEmpty()`. The earlier
   `ambients.isEmpty() && files.isEmpty()` branch already rules out the empty-files
   case, so `files.isNotEmpty()` was always true when reached. Behavior unchanged.
